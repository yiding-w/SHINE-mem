# Evaluation Export LoRA — summary.json Metrics Reference

## Overview

`summary.json` is generated by the `evaluation_export_lora` mode. It evaluates each repo's trajectories by comparing the model's loss (with LoRA adapters applied) against the baseline loss (raw LLM without any LoRA).

---

## Top-level Fields

| Field | Description |
|-------|-------------|
| `experiment` | The experiment name (from `--name` argument) |
| `dataset` | The dataset name used for evaluation |
| `max_trajectories_per_repo` | Maximum number of trajectories evaluated per repo (from `--export_lora_max_traj`) |

---

## Global Metrics (`global_metrics`)

These are averages computed across all repos (each repo contributes equally regardless of trajectory count).

| Metric | Formula | Description |
|--------|---------|-------------|
| `num_repos` | — | Total number of repos evaluated |
| `total_trajectories` | — | Sum of trajectories across all repos |
| `avg_loss` | mean of per-repo `avg_loss` | Average cross-entropy loss with LoRA adapters applied |
| `avg_baseline_loss` | mean of per-repo `avg_baseline_loss` | Average cross-entropy loss on raw LLM (no LoRA) |
| `avg_ppl` | mean of per-repo `avg_ppl` | Average perplexity with LoRA adapters (`exp(loss)`) |
| `avg_baseline_ppl` | mean of per-repo `avg_baseline_ppl` | Average perplexity on raw LLM (`exp(baseline_loss)`) |
| `avg_loss_ratio` | mean of per-repo `avg_loss_ratio` | Average of `loss / baseline_loss` per trajectory. <1 means LoRA improves loss |
| `avg_ppl_ratio` | mean of per-repo `avg_ppl_ratio` | Average of `ppl / baseline_ppl` per trajectory. <1 means LoRA improves perplexity |
| `avg_win_rate` | mean of per-repo `win_rate` | Average fraction of trajectories where LoRA loss < baseline loss |

---

## Per-repo Metrics (`per_repo.<repo_name>`)

Each repo has the following metrics computed over its trajectories (indexed by trajectory order):

### Basic Statistics

| Metric | Formula | Description |
|--------|---------|-------------|
| `num_trajectories` | — | Number of trajectories evaluated for this repo |
| `avg_loss` | `mean(per_traj_loss)` | Mean loss across trajectories (with LoRA) |
| `avg_baseline_loss` | `mean(per_traj_baseline_loss)` | Mean loss across trajectories (raw LLM) |
| `avg_ppl` | `exp(avg_loss)` | Perplexity with LoRA |
| `avg_baseline_ppl` | `exp(avg_baseline_loss)` | Perplexity without LoRA |

### Ratio Metrics

| Metric | Formula | Description |
|--------|---------|-------------|
| `avg_loss_ratio` | `mean(loss_i / baseline_loss_i)` | Average per-trajectory loss ratio. **<1 = LoRA helps** |
| `avg_ppl_ratio` | `mean(ppl_i / baseline_ppl_i)` | Average per-trajectory perplexity ratio. **<1 = LoRA helps** |
| `win_rate` | `count(loss_i < baseline_loss_i) / N` | Fraction of trajectories where LoRA outperforms baseline |

### Trend Slopes (Linear Regression over Trajectory Index)

These measure how a metric changes as trajectory index increases (i.e., as more data is accumulated). Computed via ordinary least squares: `y = slope * x + intercept`.

| Metric | What it fits | Interpretation |
|--------|-------------|----------------|
| `loss_trend_slope` | `loss` vs trajectory index | Negative = loss decreases with more trajectories (LoRA improves) |
| `ppl_trend_slope` | `ppl` vs trajectory index | Negative = perplexity decreases with more trajectories |
| `baseline_loss_trend_slope` | `baseline_loss` vs trajectory index | Should be ~0 (baseline doesn't change); non-zero indicates data ordering effects |
| `baseline_ppl_trend_slope` | `baseline_ppl` vs trajectory index | Same as above but for perplexity |
| `loss_ratio_trend_slope` | `loss_ratio` vs trajectory index | Negative = LoRA's relative advantage grows with more trajectories |
| `ppl_ratio_trend_slope` | `ppl_ratio` vs trajectory index | Negative = LoRA's relative PPL advantage grows with more trajectories |

---

## Interpretation Guide

- **loss_ratio < 1**: The LoRA adapter reduces loss compared to the raw model. Lower is better.
- **ppl_ratio < 1**: The LoRA adapter reduces perplexity. Lower is better.
- **win_rate = 1.0**: LoRA beats baseline on every single trajectory.
- **Negative trend slopes on ratio metrics**: The LoRA advantage is growing as more trajectories are accumulated (the adapter gets relatively better with more data).
- **Negative trend slopes on loss/ppl**: The absolute loss/ppl is decreasing over trajectories.

---

## File Locations

- Per-repo detailed metrics: `save/<name>/<dataset_name>/<repo_name>/eval_metrics.json`
- Global summary: `save/<name>/<dataset_name>/summary.json`
